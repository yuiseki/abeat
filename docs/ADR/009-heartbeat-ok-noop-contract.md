# ADR 009: `HEARTBEAT_OK` No-Op Contract

## Status
Accepted

## Context
Heartbeat-style jobs often have nothing actionable to report.
Without a no-op contract, periodic runs generate noisy logs and ambiguous outputs.

OpenClaw and related projects use `HEARTBEAT_OK` as a practical no-op marker.
Aligning with that convention improves conceptual compatibility and lowers migration friction.

## Decision
Use `HEARTBEAT_OK` as the canonical no-op token for `abeat`.

### 1. Default token
- Canonical no-op token: `HEARTBEAT_OK`
- Jobs may override `no_op_token` if needed, but `HEARTBEAT_OK` is the default and recommended standard

### 2. MVP detection rule
A run is treated as no-op when the first line or last line of agent output equals the configured no-op token.

### 3. Prompt contract
Context and prompts generated by `abeat` must clearly state:
- when to return `HEARTBEAT_OK`
- that no-op responses should be short and deterministic

### 4. Future hardening (not MVP)
- `ack_max_chars` for compact acknowledgements
- normalized output hashing for duplicate suppression

## Consequences
- Simple, low-cost no-op detection aligned with OpenClaw conventions
- Better signal-to-noise ratio in periodic runs and logs
- Requires prompt discipline and adapter parsing consistency
